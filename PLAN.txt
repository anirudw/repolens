1. THE CI/CD INTEGRATION: THE "--warn" FLAG
--------------------------------------------------------------------------------
Problem: A blunt `--check --max-instability` flag will fail CI/CD pipelines
unnecessarily. Files like orchestrators or dependency-injection containers are 
*supposed* to have high instability (I=1.0) because they import everything.

Solution: Implement a `--warn` flag for GitHub Actions/GitLab CI.

How it works:
- Instead of exiting with `process.exit(1)` (which blocks the merge), the CLI 
  exits with `process.exit(0)` but outputs a formatted warning.
- Delta Checking: In CI/CD, the CLI should ideally compare the Instability 
  score of the PR branch against the `main` branch. 
- Output: "Warning: This PR increases the instability of src/core/Auth.ts 
  from 0.2 to 0.6. Ensure you are not coupling core business logic."
- This allows Repolens to act as an automated Senior Architect that leaves 
  code-review comments, rather than a gatekeeper that blocks deployments.


2. THE VISUALIZATION PLAN
--------------------------------------------------------------------------------
Problem: Terminal outputs (even with picocolors) are limited. Complex 
architectures with circular dependencies need to be seen to be understood.

Path A: The Local Server (`repolens . --serve`)
- Goal: Spin up a local web server (e.g., localhost:3000) directly from the CLI.
- Tech: Serve a static HTML file bundling a library like `vis-network` or `d3.js`.
- Experience: The user's browser opens to a 2D/3D force-directed graph. 
  Nodes represent files, edges represent imports/implementations.
- Integration: Circular dependencies detected by the DFS algorithm are highlighted 
  in glowing red, showing the exact loop visually.

Path B: The Markdown Exporter (`repolens . --export=mermaid`)
- Goal: Allow developers to paste their architecture into GitHub/Notion.
- Tech: Translate the in-memory Directed Graph into a Mermaid.js string.
- Experience: CLI outputs a code block. User drops it into their PR description 
  or README.md, and GitHub natively renders it as a flowchart.
